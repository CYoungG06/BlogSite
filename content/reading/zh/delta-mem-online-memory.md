## 摘要

大型语言模型越来越需要在长期助手与 agent 系统中积累并复用历史信息。单纯扩大上下文窗口成本高昂,且往往无法保证上下文的有效利用。我们提出 **δ-mem**:一种轻量级记忆机制,在冻结的全注意力基座旁边,维护一个紧凑的**联想记忆在线状态**(online state of associative memory)。δ-mem 把过去的信息压缩进一个由 delta 规则学习的定尺寸状态矩阵,并在生成时用其读出信号,生成对基座注意力计算的低秩修正。仅凭一个 $8\times8$ 的在线记忆状态,δ-mem 把平均分提升到冻结基座的 $1.10\times$、最强非 δ-mem 记忆基线的 $1.15\times$;在记忆密集基准上提升更大——MemoryAgentBench 达 $1.31\times$,LoCoMo 达 $1.20\times$,同时基本保持通用能力。这些结果表明:不微调主干、不替换架构、不扩展显式上下文,仅用一个与注意力计算直接耦合的紧凑在线状态,就能实现有效的记忆。

## 1 引言

随着 LLM 越来越多地部署在需要持续交互的记忆密集场景——如长期个性化助手和长时程 agent 系统——它们的生命周期必须超越「回应孤立 prompt」,而是在长程记忆密集任务中持续积累、更新和复用历史信息。在这些场景里,模型表现不仅取决于理解当前输入,还取决于在测试时有效利用相关的过去上下文。

一个直觉的做法是扩大输入上下文、保留更多交互历史。但这只是把记忆问题降级为长上下文处理问题,既昂贵又越来越难驾驭:一方面,标准注意力的成本随上下文长度平方增长;另一方面,单纯加大窗口并不保证有效利用——上下文变长后,模型常出现上下文退化或「上下文腐烂」(context rot),意味着即使百万 token 的窗口也没有从根本上解决记忆问题。这些局限呼唤更高级的记忆机制:在给定上下文窗口内更紧凑地表示历史信息、跨交互动态维护它,并在测试时让基座模型能有效地使用它。

从统一的视角看,现有记忆机制可以沿两个维度刻画:**记忆状态**(历史信息如何存储)与**记忆引导**(存储的信息如何影响基座推理)。在此框架下,先前方法落入三种范式:

- **文本记忆机制(TMM)**:以文本形式存储记忆并通过输入上下文注入(MemGPT、MemoryBank、LLMLingua、mem0 等)。无需改动架构、灵活,但受限于上下文窗口、检索噪声和不可避免的压缩损失。
- **外通道记忆机制(OMM)**:把记忆放在外部模块,通过检索或编码在外部通路与基座交互(Memorizing Transformers、LongMem、MLP Memory)。模块化好,但带来额外开销、集成复杂度,以及与基座的潜在错位。
- **参数化记忆机制(PMM)**:把记忆编码进前缀或适配器参数(LoRA、Prefix-Tuning、ROME/MEMIT)。高效且兼容冻结基座,但其静态本质难以适应动态演进的信息。

综上,这些局限共同指向一个需求:一种既能维护紧凑且动态演进的记忆状态,又能通过与基座内部注意力计算紧密对齐的通路来引导基座的记忆机制。

出于这一动机,我们提出 **δ-mem**:在冻结的全注意力基座旁维护一个紧凑、动态更新的记忆。δ-mem 不把全部历史 token 留在输入上下文里,而是把过去信息压缩进**联想记忆在线状态(OSAM)**;随着新 token 到达,该状态通过 **delta 规则学习**持续更新,使模型能在一个定尺寸的矩阵表示中维护有用的历史信息。生成时,δ-mem 不是从记忆中检索文本,而是用当前输入**查询在线状态**,提取与上下文相关的联想记忆信号,再将其转化为对基座注意力组件的**低秩修正**——联想记忆由此直接参与基座的前向计算,而基座保持冻结。每次交互后在线状态继续更新,使 δ-mem 的联想记忆随时间演进。

我们在记忆密集基准(HotpotQA、LoCoMo、MemoryAgentBench)和通用能力基准(IFEval、GPQA-Diamond)上评估 δ-mem:仅凭固定的 $8\times8$ 在线状态,δ-mem 将最终平均分提升为冻结基座的 $1.10\times$,并超过最强非 δ-mem 记忆基线 $1.15\times$。记忆密集任务上提升更大:MemoryAgentBench 超过 $1.31\times$,LoCoMo 超过 $1.20\times$,TTL 子任务从 26.14 近乎翻倍到 50.50。这些结果表明:当紧凑的在线状态与注意力计算直接耦合时,无需扩展显式上下文或重型外部检索模块,也能提供有效的联想记忆。

**贡献总结**:

1. 提出 δ-mem:为冻结全注意力基座配备紧凑的联想记忆在线状态,使历史信息被动态维护并直接耦合进基座的注意力计算。
2. 证明极小的记忆状态($8\times8$ 矩阵)即可通过 OSAM 保留有用的历史信号,即使在显式历史被移除后,也能帮助模型恢复上下文相关信息。
3. 在多个记忆密集与通用能力基准上验证了显著提升,且无需全量微调或替换基座架构。

## 2 预备知识

对用于序列建模的 Transformer,记 $\mathbf x\in\mathbb{R}^{N\times d}$ 为所选层的输入隐状态序列,$N$ 为序列长度、$d$ 为隐维;单位置隐状态记为列向量 $\mathbf x_t\in\mathbb{R}^{d}$。用 $\mathbf Q,\mathbf K,\mathbf V$ 表示注意力中的 query、key、value,用 $\mathbf S_t$ 表示处理位置 $t$ 之后的在线状态。除非特别说明,下文省略层序号。

具体地,δ-mem 维护一个矩阵 $\mathbf{S}$ 作为联想记忆在线状态。随着 token 被处理,该状态被顺序更新,以紧凑地编码历史上下文中的键-值关联。给定位置 $t$ 的记忆键 $\mathbf k_t\in\mathbb{R}^{r}$ 与记忆值 $\mathbf v_t\in\mathbb{R}^{r}$,状态应存储关联 $\mathbf k_t\mapsto \mathbf v_t$。上一状态做出的预测为

$$\hat{\mathbf v}_t = \mathbf S_{t-1}\mathbf k_t .$$

这一记忆更新可以视为用 SGD 优化一个在线回归损失:

$$\mathcal{L}_t(\mathbf{S}) = \frac{1}{2}\left\|\mathbf{S}\mathbf{k}_t - \mathbf{v}_t\right\|^2,\quad \mathbf{S}_t = \mathbf{S}_{t-1} - \beta_t \nabla_{\mathbf{S}_{t-1}} \mathcal{L}_t(\mathbf{S}_{t-1}) = \mathbf{S}_{t-1} + \beta_t \left(\mathbf{v}_t-\mathbf{S}_{t-1}\mathbf{k}_t\right)\mathbf{k}_t^{\top},$$

该形式只沿 key 方向写入**残差信息**:已学好的关联几乎不引发更新,而预测误差会动态修正记忆状态。受 Qwen-Next 中门控保持(gated retention)设计的启发,我们进一步引入遗忘门来控制长程状态演化:

$$\mathbf S_t = \lambda_t\mathbf S_{t-1} + \beta_t (\mathbf v_t-\mathbf S_{t-1}\mathbf k_t) \mathbf k_t^\top .$$

其中 $\lambda_t$ 控制保留多少旧记忆,$\beta_t$ 控制残差写入的强度。这一门控 delta 更新构成了 δ-mem 稳定在线记忆动态的基础。

> **译注**:眼熟的话——这正是 DeltaNet/线性注意力一族的更新规则:记忆是一个矩阵态,按 delta 规则做在线梯度下降。本文的新意不在规则本身,而在把它用作「冻结 LLM 的外挂记忆」:状态读写都围绕注意力修正来设计。

## 3 δ-mem

![图 1:δ-mem 总览。给定冻结 Transformer 基座的隐状态,δ-mem 先将其投影到低维记忆空间,从上一在线状态读出与上下文相关的联想记忆信号,用该信号生成对注意力计算的低秩修正;计算完成后,再通过 delta 规则把当前的键值信息写入记忆状态。](/images/reading/delta-mem/pipeline.png)

在每个位置,δ-mem 遵循相同的计算顺序:先从旧状态读出联想记忆信号,用信号引导注意力,再把当前信息写入状态。这样,模型可以把历史压缩进一个随序列演化的状态并在后续推理中使用,而无需更新基座参数。图 1 概括了这一设计:冻结基座执行标准注意力计算;δ-mem 从上一状态读出、生成 query 侧与输出侧的注意力修正,并用当前记忆键值更新在线状态。图中还总结了本文研究的三种写入策略:token 级更新、段级更新与多状态记忆组织。

### 3.1 记忆投影

为形成联想记忆在线状态,给定当前位置的隐状态 $\mathbf x_t\in\mathbb{R}^{d}$,δ-mem 将其投影到低维联想记忆空间:

$$\mathbf q_t^m = L_2\normop \left( \tanh(\mathbf W_q^m \mathbf x_t) \right), \quad \mathbf k_t^m = L_2\normop \left( \tanh(\mathbf W_k^m \mathbf x_t) \right), \quad \mathbf v_t^m = \mathbf W_v^m \mathbf x_t,$$

其中 $\mathbf q_t^m,\mathbf k_t^m,\mathbf v_t^m\in\mathbb{R}^{r}$ 分别对应记忆的读与写:$\mathbf q_t^m$ 查询旧状态,$\mathbf k_t^m$ 与 $\mathbf v_t^m$ 描述当前信息应如何写入状态。对 query 和 key 做归一化,可以降低长序列递归中尺度漂移带来的状态不稳定。

写入门(write gate)与保持门(retention gate)同样由当前隐状态决定:

$$\boldsymbol\beta_t=\sigma(\mathbf W_\beta \mathbf x_t+\mathbf b), \quad \boldsymbol\lambda_t=\mathbf 1-\boldsymbol\beta_t,$$

其中 $\boldsymbol\beta_t,\boldsymbol\lambda_t\in\mathbb{R}^{r}$,$\sigma$ 为 sigmoid。这使得状态更新可以逐维调节:一些维度保留旧记忆,另一些维度更积极地写入当前信息。

### 3.2 从联想记忆在线状态读取

在写入当前信息之前,δ-mem 先从旧状态读取:

$$\mathbf r_t = \mathbf S_{t-1}\mathbf q_t^m .$$

读向量 $\mathbf r_t\in\mathbb{R}^{r}$ 是用当前输入查询在线记忆状态的结果。由于 $\mathbf S_{t-1}$ 大小固定,这一步的开销与历史长度无关。

这种读取形式与标准注意力是互补的:注意力把 query 与显式上下文中的所有 key 比较,而 δ-mem 直接从压缩状态获得连续的联想记忆信号——它不返回文本片段、不增加上下文 token,而是在注意力计算之前提供历史依赖的引导信号。

### 3.3 通过低秩修正引导注意力

联想记忆信号通过两个轻量线性映射引导注意力计算。首先,读出信号 $\mathbf r_t$ 被投影为 query 侧修正和输出侧修正:

$$\Delta \mathbf q_t = \mathbf W_q^\Delta \mathbf r_t, \qquad \Delta \mathbf o_t = \mathbf W_o^\Delta \mathbf r_t .$$

query 侧修正被加到冻结基座的原始 query 上:

$$\mathbf q_t^0 = \mathbf W_Q \mathbf x_t, \qquad \tilde{\mathbf q}_t = \mathbf q_t^0+\frac{\alpha}{r}\Delta \mathbf q_t .$$

随后用修正后的 query 与冻结基座的 key、value 计算注意力输出 $\mathbf a_t$,输出侧修正加在注意力之后:

$$\mathbf a_t = \Attn(\tilde{\mathbf q}_t,\mathbf K_{\leq t},\mathbf V_{\leq t}), \qquad \tilde{\mathbf y}_t = \mathbf a_t+\frac{\alpha}{r}\Delta \mathbf o_t .$$

主实现只在 query 侧和输出侧使用这两个修正项(选择依据见消融一节)。这里的低秩修正不同于静态 adapter:虽然 $\mathbf W_q^\Delta$ 和 $\mathbf W_o^\Delta$ 训练后固定,但它们的输入 $\mathbf r_t$ 来自动态状态 $\mathbf S_{t-1}$——同一组参数在不同历史下会产生不同的引导效果。

### 3.4 写入联想记忆在线状态

当前注意力计算完成后,δ-mem 把当前位置的信息写入在线状态。给定当前记忆键值对 $(\mathbf k_t^m,\mathbf v_t^m)$,上一状态先预测当前 key 对应的值 $\mathbf S_{t-1}\mathbf k_t^m$;目标值与该预测之差,即为要写入的残差信息。δ-mem 用逐维门控的 delta 规则更新状态:

$$\mathbf S_t = \mathrm{Diag}(\boldsymbol\lambda_t)\mathbf S_{t-1} + \mathrm{Diag}(\boldsymbol\beta_t) \left( \mathbf v_t^m-\mathbf S_{t-1}\mathbf k_t^m \right) (\mathbf k_t^m)^\top .$$

展开得:

$$\mathbf S_t = \mathrm{Diag}(\boldsymbol\lambda_t)\mathbf S_{t-1} - \mathrm{Diag}(\boldsymbol\beta_t) \mathbf S_{t-1} \mathbf k_t^m(\mathbf k_t^m)^\top + \mathrm{Diag}(\boldsymbol\beta_t) \mathbf v_t^m(\mathbf k_t^m)^\top .$$

三项各有明确角色:第一项保留旧状态;第二项沿当前 key 方向移除旧的预测成分;第三项把新值写入同一方向。因此,记忆状态的更新是**受控遗忘下的误差修正**,而不是不加选择地累积新的外积。

逐维门控可以从按行展开看出。记 $\mathbf s_t^{(i)}$ 为 $\mathbf S_t$ 的第 $i$ 行:

$$\mathbf s_t^{(i)} = \lambda_{t,i}\mathbf s_{t-1}^{(i)} + \beta_{t,i} \left( v_{t,i}^m-\mathbf s_{t-1}^{(i)}\mathbf k_t^m \right) (\mathbf k_t^m)^\top .$$

即每个记忆维度可以独立控制保留多少旧信息、写入多强的当前残差。这种逐维门控对持续交互很有用:状态既要保住稳定的历史信息,又要及时吸收新输入。

### 3.5 在线状态的写入粒度

上述公式解释了单次写入如何进行,但记忆机制还取决于写入粒度的定义。token 是最细的粒度,但并不总是最合适的:在对话和 agent 轨迹中,消息、语义段或阶段级事件往往更稳定。我们考察三种写入策略(见图 1):TSW 每个 token 写一次;SSW 对每个段内隐状态取平均后按段写入;MSW 写入多个并行的子状态再聚合读出。

**Token-State Write(TSW)**:在每个 token 位置更新在线状态 $\mathbf S_t = \Update(\mathbf S_{t-1},\mathbf x_t)$。保留最细粒度信息,适合需要捕捉局部变化的场景;但每个 token 都触发写入,状态也更容易受格式符号、重复表达和短期噪声影响。

**Sequence-State Write(SSW)**:把写入粒度从单个 token 提升到消息段。记 $\mathcal{M}^{(j)}$ 为第 $j$ 条消息的 token 索引集合,先对段内全部 token 的隐状态取平均得到段表示 $\bar{\mathbf x}^{(j)} = \frac{1}{|\mathcal{M}^{(j)}|} \sum_{t \in \mathcal{M}^{(j)}} \mathbf x_t$,然后每条消息更新一次状态:$\mathbf S_{(j)} = \Update(\mathbf S_{(j-1)},\bar{\mathbf x}^{(j)})$。SSW 减少冗余写入、平滑状态演化;代价是一些细粒度 token 级细节被平均掉了。

**Multi-State Write(MSW)**:前两种策略调整写入粒度,MSW 调整的是状态组织。单一状态需要同时容纳事实、偏好、任务进展和局部事件,容易发生覆写与干扰。MSW 把记忆分解为多个并行子状态:

$$\mathcal S_t=\{\mathbf S_t^{(1)},\ldots,\mathbf S_t^{(N)}\},\qquad \mathbf S_t^{(i)}=\Update^{(i)}(\mathbf S_{t-1}^{(i)},\mathbf x_t),\qquad \mathbf r_t=Concat(\mathbf r_t^{(1)},\ldots,\mathbf r_t^{(N)}),$$

其中 $\mathbf S_t^{(i)}\in\mathbb{R}^{r\times r}$,$\mathbf r_t^{(i)}=\mathbf S_{t-1}^{(i)}\mathbf q_t^{m,(i)}$。这种组织让不同子状态积累不同类型的信息,减少单一状态内部的相互干扰。

### 3.6 训练目标

δ-mem 用标准 SFT 损失训练。对每个样本:上下文 token 先被写入在线状态得到 $\mathbf S_C$,预测时它们**不再**作为显式输入回放给基座;冻结基座只接收 query $Q$ 与 response $Y$,而存储的状态通过 δ-mem 引导注意力。损失为 response token 上的自回归交叉熵:

$$\mathcal L_{\mathrm{SFT}} = -\sum_{j=1}^{|Y|} \log p_{\phi,\theta}(y_j \mid Q, y_{<j}, \mathbf S_C),$$

其中 $\theta$ 为可训练的 δ-mem 参数,$\phi$ 为冻结的基座参数。

> **译注**:这个训练设定很关键——它强制「记忆只能靠状态传递」:上下文不进 prompt,答案要答对,模型就必须学会把有用信息写进状态、再从状态读出来。这也解释了后面「无上下文恢复」实验为什么成立。

## 4 实验

### 4.1 实验设置

**评估与基准。** 为分别衡量通用推理与记忆效果,我们在通用任务与记忆密集基准上评估:通用多跳推理、知识密集 QA 与指令遵循用 HotpotQA、GPQA-Diamond、IFEval;记忆密集侧用 LoCoMo(按 mem0 的做法排除对抗性问题类)与 MemoryAgentBench(评估跨长交互历史的记忆保持、检索与利用)。

**基线。** 所有方法都基于同一个 Qwen3-4B-Instruct 基座。文本记忆侧:BM25 RAG(检索相关历史文本拼入上下文)、LLMLingua-2(把长历史压缩为更短的文本上下文)、MemoryBank(以文本条目维护连续交互历史);参数化记忆侧:Context2LoRA、MemGen(把记忆或上下文相关适配编码进额外可训练参数);外通道记忆侧:MLP Memory(在独立模块检索后融合回模型)。附录中还报告了 rank-8 配置下的可训练参数量对比。

**实现细节。** 基座涵盖 Qwen3-8B、Qwen3-4B-Instruct、SmolLM3-3B;全部模型在 QASPER 最短的 2,219 样本划分上训练 1 epoch(最长序列 8,269 token),基座训练序列长 512,记忆写入预算 8,192 token。默认 $r=8$、$\alpha=16$,只作用于 query 与 output 分支;MSW 状态数为 4。8×A800、bf16、DeepSpeed ZeRO-2,峰值学习率 $2\times10^{-4}$ 余弦衰减,有效 batch 32。

### 4.2 跨记忆机制的主结果

下表对比了各记忆机制在 Qwen3-4B-Instruct 上的表现(为清晰起见仅列关键指标,完整子指标见原文):

| 方法 | IFEval | HotpotQA EM/F1 | GPQA | MAB 平均 | MAB-TTL | LoCoMo 平均 | 总平均 |
|---|---|---|---|---|---|---|---|
| Qwen3-4B-Instruct | 81.89 | 42.35 / 56.00 | 39.39 | 29.54 | 26.14 | 40.79 | 46.79 |
| + BM25 RAG | - | 40.35 / 52.83 | - | 24.49 | 9.74 | 36.68 | 44.56 |
| + LLMLingua-2 | - | 36.93 / 50.03 | - | 15.63 | 1.43 | 40.98 | 42.96 |
| + MemoryBank | - | - | - | 17.65 | 7.67 | 38.14 | 43.88 |
| + Context2LoRA | 76.71 | 37.85 / 50.88 | 29.29 | 32.53 | 29.86 | 48.11 | 44.90 |
| + MemGen | 39.37 | 5.36 / 16.27 | 38.89 | 29.61 | 28.45 | 40.05 | 30.66 |
| + MLP Memory | 24.95 | 10.94 / 25.83 | 22.73 | 28.80 | 26.00 | 26.85 | 22.85 |
| + δ-mem (SSW) | 81.70 | 49.22 / 63.43 | **41.41** | 37.84 | **50.50** | 47.05 | 51.44 |
| + δ-mem (TSW) | **82.99** | **49.41** / **63.66** | 40.40 | 36.48 | 40.64 | 46.53 | **51.66** |
| + δ-mem (MSW) | 81.52 | 46.86 / 60.47 | 37.37 | **38.85** | 47.29 | **49.12** | 50.74 |

δ-mem 全面最强:TSW 变体总平均 51.66%,比基座(46.79%)高 +4.87,比 Context2LoRA(44.90%)高 +6.76;SSW、MSW 也分别达到 51.44%、50.74%。**记忆密集基准上的增益最显著**:MemoryAgentBench 平均从 29.54% 提升到 38.85%(MSW 最佳);LoCoMo 上 MSW 平均最高 49.12%,并在 Multi、Temporal、Open 子集上最佳;HotpotQA 上 TSW 把 EM/F1 从 42.35%/56.00% 提升到 49.41%/63.66%。

各基线的短板也很典型:文本记忆方法增益不稳定,可能是因为把记忆压回 token 空间引入了检索噪声与信息损失;Context2LoRA 这类参数化方法的记忆静态编码在参数里,容易过拟合训练分布,跨任务泛化不稳;MLP Memory 缺乏序列状态积累、无法显式建模长程依赖,表现有限。相比之下,δ-mem 在通用与记忆密集评估中都稳定提升,说明把记忆维护为在线状态是更稳健的记忆机制。

### 4.3 跨基座模型的主结果

δ-mem 在三个基座上全面提升总平均:Qwen3-4B-Instruct 46.79% → 51.66%,Qwen3-8B 47.20% → 50.86%,SmolLM3-3B 26.08% → 36.96%。值得注意的是,**写入策略的有效性随模型能力变化**:更强的 Qwen3-8B 上 SSW 最佳(50.86%)——段级写入平滑了状态更新、抑制 token 级噪声;而较小的 SmolLM3-3B 由 MSW 驱动大幅提升(26.08% → 36.96%)——小基座更受益于把记忆拆成多个状态以减少干扰。

## 5 消融研究

### 5.1 上下文恢复(Context Recovery)

![图 2:以 Qwen3-4B-Instruct 为基座,在 HotpotQA 与 LoCoMo 上的上下文恢复表现。](/images/reading/delta-mem/context_recover.png)

为检验在线状态能否在**不回放显式上下文**的情况下保住有用的历史信息,我们在无上下文(no-context)设定下评估 δ-mem:移除原始历史上下文,只注入压缩的记忆状态。结果表明 δ-mem 在两个基准上都持续优于无上下文基线:HotpotQA 整体 EM 从 0.08% 提升到 6.48%,F1 从 8.27% 到 15.20%;多跳证据最多的 Bridge 子集提升尤其大,说明在线状态能恢复一部分缺失的多跳证据。LoCoMo 总平均也从 3.49% 提升到 8.05%,多跳、时序、开放域、单跳各类问题均有明确增益。这说明联想记忆在线状态确实存下了与上下文相关的历史信号,在显式上下文不可用时仍能复用。

> **译注**:绝对值看起来不高(EM 6.48%),但要想清楚分母:这是「全文删掉、只剩一个 8×8 矩阵」的设定。能从 0.08% 恢复到这个水平,已经证明小矩阵里真装着东西——这个实验是整个论文里最有「记忆感」的一个。

### 5.2 注入分支消融(Heads Ablation)

记忆修正应该注入注意力块的哪个分支?结果表明:query+output 双分支(qo)已经很强(平均 47.97%);单分支中 output 分支最好(47.05%),key 分支最弱;全分支 qkvo 最高(48.05%),但相对 qo 的边际提升不值得额外的参数开销,因此默认取 qo——性能与效率的最优平衡。这说明联想记忆信号在能联合影响 query 形成、key-value 交互与输出表示时最有效。

### 5.3 注入深度消融(Insertion Depth)

把记忆修正应用到**全部层**效果最好(平均 47.97%),HotpotQA 与 LoCoMo 也均最佳——联想记忆信号在能影响基座全部深度的表示层级时最有效。部分层变体中,中间层配置最好(46.66%),明显优于前层与后层配置:中间层在语义抽象与任务相关计算之间取得了平衡;前层注入作用的表示过于局部,后层注入则留给联想记忆信号向下游传播的深度不足。

## 6 相关工作

**文本记忆机制。** 这类方法把记忆外化为文本条目、摘要或可检索文档,并把选中的证据重新注入输入上下文或 RAG 流程。早期检索增强系统证明了可扩展文本库对知识密集生成的有效性;后续面向 agent 的方法(MemGPT、MemoryBank、mem0 等)通过记录、摘要与反思把这一范式扩展到持续交互。尽管灵活,文本记忆仍受制于其 token 化形态:记忆的使用对压缩保真度、检索噪声和上下文预算敏感。δ-mem 不把压缩历史引回 token 空间,而是维护紧凑的在线状态、用其读出引导冻结 Transformer 的注意力,把记忆维护与 prompt 级重插分离开来。

**外通道记忆机制。** 另一条相关工作线把记忆以潜在(而非文本)形式存储在基座之外:Memorizing Transformers 把过去的内部表示存为不可微的键值记忆并用近似 kNN 检索;LongMem 用冻结基座做记忆编码器、用自适应残差侧网络从外部记忆库读取。相比文本记忆,潜在记忆能避免一部分自然语言摘要带来的信息损失、保留更丰富的内部表示;但记忆仍通过独立的检索/读取通路与基座交互,带来检索开销、融合复杂度,以及存储表示与当前表示之间可能的错位。δ-mem 的不同在于:记忆不是作为辅助外部资源被检索,而是其紧凑在线状态直接产出对注意力计算的低秩修正,让记忆参与当前前向传播。

**参数化记忆机制。** 这类机制把记忆编码进额外参数或局部权重编辑:Prefix-Tuning 为冻结模型学习连续虚拟 token;LoRA 在选定层注入低秩可训练更新;ROME、MEMIT 等模型编辑方法进一步把参数当作可写记忆基底,通过局部或低秩权重更新插入事实关联。然而这些方法并不适合在线记忆:它们的记忆通常在训练后固定、或通过离散的编辑步骤更新,而不是随序列连续演化;写入粒度也与常以 token/消息/段级展开的交互历史不对齐。因此,参数化记忆更像对模型行为的持久修改,而非随当前历史变化的状态条件记忆。δ-mem 在低秩接口上与 LoRA 相近,但根本区别在于:LoRA 的低秩更新是静态的,而 δ-mem 的低秩注意力修正是在运行时从紧凑在线状态生成的。

## 7 结论

本文提出了 δ-mem:一种轻量级记忆机制,为冻结的全注意力基座配备紧凑且动态更新的联想记忆在线状态。δ-mem 把过去信息压缩进定尺寸的在线状态,用其读出生成对基座注意力组件的低秩修正,使记忆得以在线维护并直接参与前向计算——无需全量微调或替换基座架构。实验上,δ-mem 在基本保持冻结基座通用能力的同时提升了记忆密集基准的表现;尤其值得注意的是,即使只用极小的 $8\times8$ 在线状态,模型也能在显式上下文被移除后恢复有用的历史信息——有效的记忆并不需要扩展显式上下文或重型外部检索模块。这些结果表明:紧凑在线状态可以作为冻结 Transformer 基座上可扩展、高效的测试时记忆接口。

## 附录要点

**推理效率与显存**:δ-mem 的 GPU 显存占用与 Vanilla、Context2LoRA 几乎相同(prompt 到 32K 时在线状态的额外开销可忽略),而 MLP Memory、MemGen 显著更吃显存;解码吞吐上,δ-mem 因每步都要读写在线状态而慢于 Vanilla 与 Context2LoRA,但在所有测试设定下都明显比 MemGen 更快更稳——以很轻的计算足迹换取记忆能力。

![图 3:不同 prompt 与解码长度下的推理效率(解码吞吐与显存占用)。](/images/reading/delta-mem/tps.png)

**参数开销**:SSW/TSW 仅引入 487 万可训练参数(占基座 0.12%),多状态的 MSW 也只有 1,947 万(0.48%);对比 MemGen 的 4,620 万和 MLP Memory 的 30.78 亿(占基座 76.40%),δ-mem 以低得多的参数开销实现在线记忆增强。

## 结语评价(编者)

**亮点**:

- 视角清晰:用「记忆状态 × 记忆引导」两个维度把文本/外通道/参数化三种范式统一起来,然后各自打了一枪——综述部分本身就是很好的记忆机制地图;
- 路线便宜且「神经」:不检索、不回填、不微调主干,487 万参数换来记忆密集任务的显著提升,工程接入成本低;
- 无上下文恢复实验设计得聪明,直接证明了小矩阵里存着可用信息;三种写入粒度(TSW/SSW/MSW)的划分有实操指导意义;
- 与 LoRA 的对比点得准:接口形似,但 LoRA 静态、δ-mem 的修正随状态动态生成——这是「参数即记忆」和「状态即记忆」的分水岭。

**局限与疑点**:

- 训练数据只用了 QASPER 一个 2,219 样本的小集、1 epoch——方法在更大更多样的记忆语料上的表现还是问号;基准提升幅度可能部分来自任务对齐而非纯记忆能力;
- 基座最大只到 8B,更大模型上在线状态的容量瓶颈($8\times8$ 够不够用)没有回答;
- 上下文恢复的绝对分数仍低,说明小状态目前更像「提示器」而非「保险柜」——离真正替代显式上下文还很远;
- 与文本记忆并非非此即彼:检索+状态的混合路线(大事进文本库、近期进在线状态)可能是更实用的形态,论文未探索。

**相关阅读**:本站 [RLM 译文](/zh/distilled/lm-harness-compositional-generalizers/)的「局部在分布内」与本文互补——RLM 让每次调用落在训练分布内,δ-mem 则让历史以状态形式保持可用,两者都是「不靠扩上下文」的路线;速递区近期的 MemGPT/Mem0 类工作可作为文本记忆范式对照。
