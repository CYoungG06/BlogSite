## 摘要

LLM 正越来越多地被训练为交互式智能体,执行多轮交互、工具使用与环境反馈的长时程任务。基于结果的强化学习(RL)提供了实用的优化范式,但其稀疏的轨迹级奖励对中间决策的指导有限,在「episode 级结果」与「token 级策略学习」之间留下了监督鸿沟。我们提出 **Seed**(SElf-Evolving On-Policy Distillation,自我进化的 on-policy 蒸馏):一个把跑完的 on-policy 轨迹转化为训练时「事后技能」(hindsight skills)并将其行为效果蒸馏回策略模型的自进化框架。Seed 首先微调策略,使其能分析完整轨迹并生成自然语言技能——捕捉可复用的工作流、决定性观察或避坑规则。在 RL 阶段,当前策略既负责收集轨迹,又充当从轨迹中提取事后技能的分析器;策略更新因此同时改进后续决策与技能分析,让事后监督随策略共同进化。随后,Seed 在普通上下文与技能增强上下文下对采样动作重新打分,把技能引起的概率偏移转化为稠密的 token 级 on-policy 蒸馏信号,并与结果奖励 RL 联合优化,使辅助监督始终与当前轨迹分布对齐。在文本与视觉 agentic 任务上的大量实验表明,Seed 持续提升性能与样本效率,并对未见场景展现出稳健的泛化能力。

## 1 引言

近期的 LLM 系统正从单轮回答生成走向多轮 agentic 交互:模型反复推理、行动、调用工具并吸收环境反馈。这类设定要求 agent 做出序贯决策,其后果可能在许多步之后才显现——模型必须学会何时收集信息、何时调用工具、如何解读反馈、以及在部分进展或失败后如何修正计划。RL 因此成为 LLM agent 的重要后训练范式:它直接针对来自环境、模拟器或验证器的任务级反馈优化策略。

然而,基于结果的 agentic RL 只提供粗糙的监督。在长时程环境中,奖励往往稀疏、延迟、且只给在轨迹层面:它告诉你这一局成没成,却不告诉你哪些中间观察、动作或工具调用该被强化或纠正。这在 episode 级结果与 token 级策略学习之间留下了监督鸿沟:一条失败的轨迹可能包含有用的局部行为,只因几处局部错误而失败;一条成功的轨迹可能包含可复用的策略,标量奖励却从未指认出来。因此,仅优化结果难以为长交互历史提供细粒度、决策级的信用分配。

一个关键观察是:**跑完的轨迹蕴含着在线决策时不可得的事后信息(hindsight)**。当一局结束,完整的交互历史会揭示:哪些子目标达成了、agent 在哪里偏离了有效策略、哪些观察是决定性的、哪些行为模式可以迁移到未来的尝试。这与 RL 中的事后学习(把已完成的经验重新解释以改善稀疏反馈下的学习)以及语言 agent 的语言化反思、情景记忆、经验总结等方法一脉相承。但许多此类方法把事后信息当作静态经验、推理时上下文或可检索记忆。对实用的 agentic RL 而言,事后监督不应固定不变:随着策略改进、遇见新的状态、策略与失败模式,从轨迹中提取的事后信息必须适应当前行为。我们的目标,就是把策略自己产生的事后信息**转化为参数化的监督**,让策略把可复用的行为指导内化,而不依赖外部记忆或额外的部署时提示。

**On-policy 蒸馏(OPD)**为把事后信息转化为决策级学习信号提供了天然机制。经典知识蒸馏通过 token 或序列级监督把教师行为迁移给学生,而 on-policy 变体通过在学生策略自己采样的输出上施加监督来减少分布失配。近期的 on-policy 自蒸馏方法更进一步:不再需要外部教师,而是比较同一模型在不同上下文下的输出(比如有/无特权推理轨迹、有/无反馈条件的 prompt)。相关的 agentic 方法也探索了以技能或反馈为条件的蒸馏。综合这些进展,agentic RL 中有效的事后监督应满足三点:

1. **On-policy**:有用的纠正依赖于当前策略诱导出的状态、动作与失败模式;
2. **稠密(dense)**:轨迹级的事后信息应能指导单个决策 token,而不只是最终结果;
3. **自我进化(self-evolving)**:随着策略改进,其决策能力与轨迹分析能力应共同提升。固定的教师、静态的技能数据集和一次性蒸馏,都无法持续适配策略的能力与轨迹分布的演化。

我们据此提出 Seed:一个把跑完的 on-policy 轨迹变成事后技能、并通过自进化训练循环将其行为效果蒸馏回策略的框架。RL 阶段,当前策略既收集轨迹,又充当分析器,把轨迹提炼为自然语言技能;由于两个角色共享同一模型,策略更新同时改进决策与技能分析,事后监督随策略共同进化。给定提取出的技能,Seed 保持采样动作不变,在普通上下文与技能增强上下文下重打分,所得概率偏移构成稠密的 token 级 OPD 信号,与结果奖励 RL 目标联合优化。具体来说,Seed 包含两个阶段:第一阶段,事后技能 SFT 让模型学会分析完整交互历史并生成可复用的轨迹级技能;第二阶段,自进化 OPD 反复使用最新策略检查点同时承担轨迹收集与技能分析,用 RL+OPD 联合目标更新策略。生成的技能只在训练时充当特权监督,推理时既不需要外部记忆也不需要额外 prompt。

我们在具身交互、网页导航、搜索问答以及视觉感知与规划上评估了 Seed,取得了更优的任务表现、样本效率与鲁棒性。**贡献总结**:

1. 提出 Seed:一个自进化 OPD 框架,持续把策略跑完的轨迹转化为事后技能并在 agentic RL 中内化其行为指导,让决策与技能分析共同进化;
2. 提出策略同步的事后 OPD 机制:把技能引起的对数概率偏移转化为稠密的 token 级监督,并与结果奖励 RL 联合优化;
3. 在多个长时程 agentic 基准上的充分实验表明,Seed 在任务表现、样本效率与鲁棒性上超越代表性基线。

## 2 相关工作

**Agentic LLM 的强化学习。** LLM 正被越来越多地训练为在长时程上推理、用工具、行动的交互式智能体。RL 通过直接优化任务级结果为此类 agent 提供了自然的后训练范式。然而,基于结果的 RL 通常只在轨迹层面给出稀疏而延迟的奖励,对哪些中间观察、动作或工具调用应被强化提供的指导有限。Seed 保留结果奖励 RL 作为优化主干,同时用从跑完轨迹中导出的稠密监督加以补充。

**语言 agent 的事后学习。** 跑完的轨迹会暴露可复用的策略、决定性的观察与失败原因,这些在线决策时不可得。先前工作通过事后重标记、回报分解、过程监督、语言化反思与经验记忆来利用这类信息。然而,事后知识常被存为静态经验,或作为额外的推理时上下文引入。Seed 则把跑完的 on-policy 轨迹转化为自然语言技能,并把其行为指导内化到策略中——推理时既不需要外部记忆,也不需要技能提示。

**Agentic RL 的 on-policy 自蒸馏。** 知识蒸馏通过 token 或序列级监督迁移教师行为;on-policy 变体则在学习者自己采样的输出上训练,减少分布失配。近期方法从推理轨迹、反馈或技能构造特权「自我教师」,为 agentic RL 提供 token 级指导。然而,特权监督的来源往往是静态的、外部生成的、或独立于策略更新的——随着策略改进并遇见新状态与失败模式,这类监督会变得陈旧、与当前行为失配。Seed 通过一个自进化循环解决这一局限:最新的策略检查点同时充当 rollout 执行者与轨迹分析器,每次更新后共享检查点在两个角色上同步刷新,让决策与事后监督共同进化。

## 3 方法

![图 2:Seed 总览。阶段 1(事后技能 SFT)让策略学会从跑完的轨迹中提取事后技能;阶段 2(自进化 on-policy 蒸馏)在自进化的 agentic 循环中联合优化结果奖励 RL 与技能条件 OPD。](/images/reading/seed/x3.png)

我们提出 Seed:面向 agentic RL 的自进化 OPD 框架。出发点是这样一个观察:即使奖励稀疏,完整的 agent 轨迹通常包含丰富的事后信息——有用的行为模式、失败原因与可复用策略,这些在中间决策步并不直接可得。Seed 把这类事后信息转化为事后技能,并将其行为效果蒸馏回普通策略。

如图 2 所示,Seed 有两个训练阶段:第一阶段,事后技能 SFT 让单个策略模型学会分析完整轨迹;第二阶段,当前策略快照既收集 on-policy 轨迹、又把轨迹分析成事后技能,随后对同一批采样动作在普通与技能增强上下文下重打分,构造 token 级 OPD 信号与 RL 联合优化。推理时移除分析器,部署只需学到的策略。

### 3.1 问题形式化

我们把长时程 agentic 任务形式化为部分可观测马尔可夫决策过程:

$$\left(\mathcal{S},\mathcal{A},\mathcal{O},\mathcal{T},\Omega,\mathcal{R},\gamma\right),$$

其中 $\mathcal{S}$ 为隐状态空间,$\mathcal{A}$ 为动作空间,$\mathcal{O}$ 为观测空间,$\mathcal{T}$ 为转移核,$\Omega$ 为观测核,$\mathcal{R}$ 为奖励函数,$\gamma$ 为折扣因子。时刻 $t$,agent 收到观测 $o_{t}\in\mathcal{O}$ 并维护交互历史

$$ h_{t}=\left(o_{0},a_{0},o_{1},a_{1},\ldots,o_{t}\right), $$

其中 $a_{i}$ 为 agent 产出的文本回复或可执行动作。策略按 $a_{t}\sim\pi_{\theta}\left(\cdot\mid h_{t}\right)$ 生成下一动作。完整轨迹记为 $\tau=\left\{(o_{t},a_{t},r_{t})\right\}_{t=0}^{T-1}$,episode 级结果为 $R(\tau)\in\mathbb{R}$。在许多 agentic 环境中,$R(\tau)$ 是稀疏的、任务完成后才给出(如二元成功标志或最终任务分)。标准 RL 目标为

$$ J(\theta)=\mathbb{E}_{\tau\sim\pi_{\theta}}\left[R(\tau)\right]. $$

这一形式化暴露了长时程 agentic RL 的核心挑战:环境只给轨迹级反馈,而策略必须从分布在整个交互历史中的大量 token 级决策里学习。Seed 通过从轨迹完成后提取的事后技能导出训练时的 token 级辅助信号,弥合这一粒度鸿沟。

### 3.2 事后技能监督微调

第一阶段让策略获得「分析完整交互历史并把可复用行为指导表达为自然语言事后技能」的初始能力。

**离线轨迹收集。** 先用不带技能增强的基础策略收集离线轨迹池:对训练任务集 $\mathcal{Q}_{\mathrm{sft}}=\{q_{j}\}_{j=1}^{M}$ 中每个任务 $q_{j}$,用基础策略 $\pi_{\theta_{\mathrm{base}}}$ 跑 $K_{0}$ 条 rollout:

$$\mathcal{B}_{j}=\left\{\tau_{j,k}\right\}_{k=1}^{K_{0}},\qquad\tau_{j,k}\sim\pi_{\theta_{\mathrm{base}}}\left(\cdot\mid q_{j}\right),$$

汇成完整离线轨迹池 $\mathcal{B}=\bigcup_{j=1}^{M}\mathcal{B}_{j}$。每条轨迹包含任务描述、观测、动作、奖励与最终结果;全部在不带事后技能的普通决策上下文中收集,保证 SFT 数据来自普通的 agent-环境交互。

**事后技能标注。** 给定一条跑完的轨迹 $\tau$,外部分析器 $A_{\mathrm{ext}}$ 产出一则事后技能标注:

$$ s_{\tau}=A_{\mathrm{ext}}\left(\tau\right),$$

其中 $s_{\tau}$ 为从轨迹 $\tau$ 产出的技能标注:成功轨迹的 $s_{\tau}$ 通常捕捉促成任务完成的可复用策略或工作流;失败轨迹的 $s_{\tau}$ 则可编码从观察到的失败中推断出的纠正或规避指导。只有格式正确的标注才被保留(记有效性指示 $v_{\tau}\in\{0,1\}$),得到 SFT 集合

$$\mathcal{D}_{\mathrm{sft}}=\left\{(x_{\tau},s_{\tau}):\tau\in\mathcal{B},v_{\tau}=1\right\},$$

其中 $x_{\tau}$ 为序列化的轨迹分析输入,$s_{\tau}$ 为对应的事后技能目标。

**监督微调。** 随后微调策略模型,使其能从完整轨迹预测事后技能——同一个自回归模型之后会同时初始化 RL 执行者与同步轨迹分析器。对被接受的 SFT 样本 $(x_{\tau},s_{\tau})\in\mathcal{D}_{\mathrm{sft}}$,用标准负对数似然目标优化:

$$\mathcal{L}_{\mathrm{sft}}(\theta)=-\mathbb{E}_{(x_{\tau},s_{\tau})\sim\mathcal{D}_{\mathrm{sft}}}\left[\sum_{\ell=1}^{|s_{\tau}|}\log\pi_{\theta}\!\left(s_{\tau,\ell}\mid x_{\tau},s_{\tau,<\ell}\right)\right],$$

其中 $s_{\tau,\ell}$ 为 $s_{\tau}$ 的第 $\ell$ 个 token。所得检查点 $\theta_{\mathrm{sft}}$ 初始化后续 RL 策略。RL 阶段,轨迹分析器直接由当前策略检查点实例化——同一个模型既能在普通交互历史下行动,也能从跑完的轨迹生成事后技能,无需单独训练分析器。

### 3.3 自进化 On-Policy 蒸馏

第二阶段进行带 token 级 OPD 附加信号的 agentic RL。每次更新开始时,Seed 冻结当前策略为 $\pi_{\theta_{\mathrm{old}}}$:该快照负责收集轨迹,也参数化了提取事后技能的分析器。可训练策略 $\pi_{\theta}$(由 $\pi_{\theta_{\mathrm{old}}}$ 初始化)随后用环境驱动的 GRPO 目标与 OPD 目标共同优化;更新完成后,$\pi_{\theta}$ 成为下一轮迭代的策略快照。这一设计保证蒸馏信号始终在策略上:策略收集自己的轨迹,分析器把跑完的轨迹转化为事后技能,事后指导的行为效果再被蒸馏回普通策略。

**On-policy 事后技能生成。** 对每个任务 prompt $q$,Seed 用冻结策略采样一组 $N$ 条轨迹:

$$\mathcal{G}_{q}=\left\{\tau_{q}^{(1)},\tau_{q}^{(2)},\ldots,\tau_{q}^{(N)}\right\},\qquad\tau_{q}^{(n)}\sim\pi_{\theta_{\mathrm{old}}}\left(\cdot\mid q\right).$$

对每条跑完的轨迹 $\tau_{q}^{(n)}$,构造其轨迹分析输入 $x_{\tau_{q}^{(n)}}$;由同一策略快照 $\pi_{\theta_{\mathrm{old}}}$ 实例化的分析器 agent 分析该轨迹并生成事后技能:

$$ s_{q}^{(n)}=A_{\theta_{\mathrm{old}}}\left(x_{\tau_{q}^{(n)}}\right),$$

其中 $A_{\theta_{\mathrm{old}}}$ 为共享模型的分析器角色。尽管执行者与分析器在每次更新时共享参数,角色不同:执行者与环境交互,分析器把跑完的轨迹总结为轨迹级事后技能。这种共享参数化构成了 Seed 的自进化循环:刷新 $\theta_{\mathrm{old}}$ 同时改变执行者遇到的轨迹分布和用于技能分析的模型能力——经验分布与其事后监督共同进化。

**On-policy 蒸馏目标。** Seed 保持原始 on-policy 动作不变,在技能增强上下文下对其重打分。记 $H$ 为确定性的上下文增强函数,把生成的技能并入普通交互历史:轨迹 $\tau_{q}^{(n)}$ 中时刻 $t$ 的技能增强历史为

$$\tilde{h}_{q,n,t}=H\left(h_{q,n,t},s_{q}^{(n)}\right).$$

原始采样动作 tokenize 为 $a_{q,n,t}=\left(a_{q,n,t,1},\ldots,a_{q,n,t,L_{q,n,t}}\right)$。同一策略对同一批动作 token 计算两组 token 级对数概率——**技能条件教师分支**(在技能增强历史下重打分):

$$\ell^{\mathrm{skill}}_{q,n,t,\ell}=\log\pi_{\theta}\left(a_{q,n,t,\ell}\mid\tilde{h}_{q,n,t},a_{q,n,t,<\ell}\right),$$

与**普通学生分支**(在原始交互历史下):

$$\ell^{\theta}_{q,n,t,\ell}=\log\pi_{\theta}\left(a_{q,n,t,\ell}\mid h_{q,n,t},a_{q,n,t,<\ell}\right).$$

两个分支共享 $\pi_{\theta}$ 但输入上下文不同:教师看到事后技能,学生只看普通历史。教师提供**脱离梯度**(detached)的训练时信号,梯度只流经普通学生分支。优化时两分支都用当前可训练策略 $\pi_{\theta}$ 求值,而 $\pi_{\theta_{\mathrm{old}}}$ 用于轨迹收集、技能生成与重要性比率计算。

定义脱离梯度的事后技能对数概率偏移:

$$\Delta_{q,n,t,\ell}=\operatorname{sg}\left[\ell^{\mathrm{skill}}_{q,n,t,\ell}-\ell^{\theta}_{q,n,t,\ell}\right],$$

其中 $\operatorname{sg}[\cdot]$ 为 stop-gradient。仿照 SDAR(Lu et al., 2026a),Seed 把该偏移映射为置信门:

$$ g_{q,n,t,\ell}=\sigma\left(\beta_{\mathrm{opd}}\Delta_{q,n,t,\ell}\right),$$

其中 $\sigma(\cdot)$ 为 sigmoid,$\beta_{\mathrm{opd}}$ 控制门的锐度。偏移为正说明事后技能支持该采样 token,门更大;偏移为负则削弱该 token 的辅助监督。

OPD 损失定义为置信门控的采样 token 蒸馏目标:

$$\mathcal{L}_{\mathrm{opd}}(\theta)=\mathbb{E}_{q,n,t,\ell}\left[m_{q,n,t,\ell}\cdot g_{q,n,t,\ell}\cdot\left(\operatorname{sg}\left[\ell^{\mathrm{skill}}_{q,n,t,\ell}\right]-\ell^{\theta}_{q,n,t,\ell}\right)\right], \tag{1}$$

其中 $m_{q,n,t,\ell}\in\{0,1\}$ 为有效 token 掩码。由于门与教师对数概率都是脱离梯度的,教师项对 $\theta$ 为常数,于是

$$\nabla_{\theta}\mathcal{L}_{\mathrm{opd}}=-\mathbb{E}_{q,n,t,\ell}\left[m_{q,n,t,\ell}\cdot g_{q,n,t,\ell}\cdot\nabla_{\theta}\ell^{\theta}_{q,n,t,\ell}\right].$$

即该目标在梯度上等价于门加权的负对数似然:最小化它会提高普通策略在教师认可的 on-policy token 上的似然,从而在不暴露技能的情况下把技能的行为效果内化。

> **译注**:这一段是全文的题眼,也是 OPD 思想的漂亮变体——「教师」不是更大的模型,而是「看到了事后技能的自己」。两分支共享同一份参数,唯一的差别是上下文里有没有技能;概率差自然度量了「技能对这个 token 的认可度」,sigmoid 门再把负偏移(技能不认可的 token)的监督压下去。推理时技能完全退场,只留一个"内化了好习惯"的普通策略。

**联合训练目标。** 除 OPD 外,Seed 还用组相对 RL 目标优化策略。对每组 $\mathcal{G}_{q}$,计算结果的均值与标准差:

$$\mu_{q}=\frac{1}{N}\sum_{n=1}^{N}R\left(\tau_{q}^{(n)}\right),\quad\sigma_{q}=\sqrt{\frac{1}{N}\sum_{n=1}^{N}\left(R\left(\tau_{q}^{(n)}\right)-\mu_{q}\right)^{2}},$$

轨迹级组相对优势为

$$ A^{\mathrm{rl}}_{q,n}=\frac{R\left(\tau_{q}^{(n)}\right)-\mu_{q}}{\sigma_{q}+\epsilon}, $$

广播到有效动作 token:$A^{\mathrm{rl}}_{q,n,t,\ell}=A^{\mathrm{rl}}_{q,n}m_{q,n,t,\ell}$。token 级概率比为

$$\rho_{q,n,t,\ell}(\theta)=\exp\left(\ell^{\theta}_{q,n,t,\ell}-\ell^{\mathrm{old}}_{q,n,t,\ell}\right),$$

RL 损失为带 clip 的替代目标加 KL 正则:

$$\mathcal{L}_{\mathrm{rl}}(\theta)=-\mathbb{E}_{q,n,t,\ell}\left[\min\left(\rho_{q,n,t,\ell}(\theta)A^{\mathrm{rl}}_{q,n,t,\ell},\operatorname{clip}\left(\rho_{q,n,t,\ell}(\theta),1-\epsilon_{\mathrm{clip}},1+\epsilon_{\mathrm{clip}}\right)A^{\mathrm{rl}}_{q,n,t,\ell}\right)\right]+\beta_{\mathrm{KL}}D_{\mathrm{KL}}.$$

最终训练目标把环境驱动的 agentic RL 与事后技能 OPD 结合:

$$\mathcal{L}_{\mathrm{SEED}}(\theta)=\mathcal{L}_{\mathrm{rl}}(\theta)+\lambda_{\mathrm{opd}}\mathcal{L}_{\mathrm{opd}}(\theta),$$

其中 $\lambda_{\mathrm{opd}}$ 控制辅助 OPD 信号的强度。RL 项优化环境结果,OPD 项内化自我生成的事后技能的效果;更新后的策略成为下一轮迭代的 $\pi_{\theta_{\mathrm{old}}}$,自进化循环闭合。

推理时,部署的 agent 只从普通交互历史行动:$a_{t}\sim\pi_{\theta}\left(\cdot\mid h_{t}\right)$。所有事后技能仅作为训练时的指导——部署不需要分析器、不需要技能库、不需要检索模块、也不需要增强的决策 prompt。

## 4 实验

### 4.1 实验设置

**基准。** 三类互补的长时程 agentic 形态:**ALFWorld**(文本具身交互,六类任务:Pick、Look、Clean、Heat、Cool、Pick2);**WebShop**(交互式网页导航,128 个标准测试任务);**搜索问答**(遵循 Search-R1 协议,先检索证据再作答,含 NQ、TriviaQA、PopQA、HotpotQA、2WikiMultiHopQA、MuSiQue、Bamboogle 七个子集)。

**基线。** 覆盖 prompting、结果奖励 RL 与蒸馏三类:Vanilla(指令微调基座,无后训练)、Skill-Prompt(只在评估上下文给技能)、GRPO(无辅助监督)、Skill-GRPO(RL 期间给策略技能条件);以及代表性自蒸馏/技能蒸馏方法:OPSD、GRPO+OPSD、Skill-SD、RLSD、SDAR。带 * 号表示评估时使用技能;其余方法在测试时都从普通交互历史运行、不使用特权技能输入。所有复现的后训练基线使用相同的基座、rollout 预算与训练日程。

**指标。** ALFWorld 报各任务族成功率及宏平均;WebShop 报归一化任务完成分与精确成功率;搜索问答报七个子集准确率及宏平均。

**实现细节。** 基座为 Qwen2.5-3B/7B-Instruct 与 Qwen3-1.7B-Instruct。SFT 阶段:每个基座采样 $M=180$ 个训练任务、每任务 $K_{0}=8$ 条轨迹(共 1,440 条),用 GLM-5.2 作为外部轨迹分析器提取事后技能,格式校验后微调 3 个 epoch。RL 阶段:SFT 检查点同时初始化策略与同步轨迹分析器,训练 150 次策略更新;ALFWorld/WebShop batch 16,搜索问答 batch 128,rollout 组大小 $N=8$。

### 4.2 主要结果

下表为主要结果(以 Qwen2.5-3B-Instruct 为例的关键列;7B 与 1.7B 趋势一致,详见原文表 1):

| 方法 | ALFWorld 平均 | 搜索 QA 平均 | WebShop 分数 | WebShop 成功率 |
|---|---|---|---|---|
| Vanilla | 21.9 | 31.7 | 6.7 | 0.8 |
| GRPO | 75.0 | 36.4 | 79.8 | 63.3 |
| Skill-GRPO* | 80.5 | 36.1 | 76.3 | 66.4 |
| GRPO+OPSD | 81.2 | 44.6 | 77.8 | 66.4 |
| Skill-SD | 73.4 | 44.1 | 75.9 | 64.0 |
| RLSD | 79.7 | 43.8 | 84.4 | 66.4 |
| SDAR | 84.4 | 43.4 | 85.0 | 68.0 |
| **Seed(本文)** | **91.8** | **45.7** | **88.5** | **78.9** |

三个发现:

**Seed 以稠密监督持续超越仅结果的 RL。** 相对 GRPO,Seed 在 ALFWorld 宏平均上提升 14.9–45.9 个百分点(三个基座),搜索 QA 提升 1.4–9.3,WebShop 分数提升 8.7–19.8,成功率提升 5.5–39.0。相对 Skill-GRPO(探索时有技能条件但仍把终局奖励广播到所有 token),Seed 进一步把 ALFWorld 平均提升 26.6–70.9 个百分点。这证明稠密的 token 级事后监督比仅优化结果提供了更有效的信用分配。

**内化技能远胜于插入提示。** 只在评估时给技能的 Skill-Prompt 在全部聚合指标上都不如 Seed;Seed 也在 12 项聚合对比中的 11 项超过 Skill-GRPO*——尽管它在评估时不使用任何技能上下文。这说明:事后技能被蒸馏进策略,比在推理时作为额外上下文提供更有效。

**自进化的事后蒸馏强于静态自蒸馏。** 在静态蒸馏基线中,Seed 在 12 项聚合对比中的 10 项取得最佳或并列最佳;优势在 ALFWorld 最明显:分别超过最强静态基线 7.4(Qwen2.5-3B)、10.2(Qwen2.5-7B)、38.1(Qwen3-1.7B)个百分点。这支持了「分析器与最新策略同步」的收益——事后监督能适应训练中遇到的轨迹与失败模式。

### 4.3 训练动力学

![图 3:ALFWorld 上的训练动力学(Qwen2.5-3B-Instruct)。半透明为原始测量,实线为 13 点居中滑动平均。](/images/reading/seed/x4.png)

图 3 对比了 Seed 与 GRPO 的优化动态:成功率曲线很早就分开——第 40 步时 Seed 约 57%,GRPO 仅约 35%,此后优势保持;Seed 的平均 episode 长度也更快下降(从约 28 轮到 13 轮,GRPO 训练结束约 16 轮)。由于更短的轨迹与更高的成功率相伴,这说明 Seed 减少了无效探索、学到了更直接的解法,而不是提前放弃任务。

### 4.4 样本效率

![图 4:样本效率分析。Seed 在各数据比例下持续优于 GRPO,仅用 60% 数据就超过全量数据的 GRPO。](/images/reading/seed/x5.png)

| 数据比例 | 20% | 40% | 60% | 80% | 100% |
|---|---|---|---|---|---|
| GRPO | 27.3 | 42.2 | 56.3 | 58.6 | 75.0 |
| Seed | 40.7 | 58.9 | **80.7** | 88.8 | 91.8 |

仅用 60% 的训练实例,Seed 达到 80.7,超过全量数据训练的 GRPO(75.0);用 40% 数据时(58.9)即与两倍数据的 GRPO(80% 数据,58.6)持平。这说明 Seed 能从每条跑完的轨迹中榨取比「只看终局奖励」更丰富的监督。

### 4.5 跨域泛化

在 ALFWorld 未见划分上评估 3B 检查点:Seed 把宏平均成功率从 70.9 提升到 86.2(超过 GRPO 15.3 个百分点),六个任务族中五个更优,提升最大的是 Heat(+35.0)、Look(+18.3)、Pick(+16.5)。说明 Seed 学到的策略学到了可迁移的行为模式,而非仅仅记住训练轨迹。

### 4.6 消融研究

| 变体 | ALFWorld 平均 |
|---|---|
| Seed | **91.8** |
| 去掉事后技能 SFT | 86.0 |
| 去掉自进化 OPD | 87.0 |
| 换成静态离线技能库 | 84.4 |

- **事后技能 SFT 的作用**:去掉后 ALFWorld 平均降 5.8 分——给执行者/分析器一个初始的轨迹分析能力是后续自进化训练的重要地基;
- **自进化 OPD 的作用**:去掉后降 4.8 分——SFT 阶段的一次性技能监督不够,必须持续从新轨迹中蒸馏才能跟上策略的进化;
- **On-policy 技能的作用**:换成静态离线技能库退化最大(-7.4)——来自当前策略自身轨迹的技能,才最贴合它不断变化的行为与失败模式。

### 4.7 定性分析

![图 6:ALFWorld 定性对比。任务「把蜡烛放进马桶」:GRPO 训练的 agent 先去目标容器、拿起无关的厕纸、随后陷入离题循环;Seed 系统地检查可能的架子、找到蜡烛,五步完成放置。](/images/reading/seed/x7.png)

图 6 展示了代表性的行为差异:GRPO 在找到所需物品前就访问目标容器、与无关物体交互、在无产出位置反复探索直到步数耗尽;Seed 则遵循更连贯的目标导向轨迹——搜索合理的储物位置、在第二个架子找到蜡烛、以正确的放置动作完成任务。这与前面的实验中「更强的表现与更高的交互效率」一致。

## 5 结论

我们提出了 Seed:一个面向长时程 agentic RL 的自进化 on-policy 蒸馏框架。Seed 通过从跑完的 on-policy 轨迹中提取事后技能、并将其行为效果蒸馏为稠密监督,弥合了稀疏轨迹级结果与 token 级策略学习之间的鸿沟。最新策略同时充当执行者与分析器,使行为与来自经验的监督共同进化,同时与当前轨迹分布保持对齐。与结果奖励 RL 的联合优化,让策略在不依赖推理时技能的情况下内化可复用的指导。在具身交互、网页导航、搜索问答、视觉推理与规划上的实验表明,Seed 在性能、样本效率与鲁棒性上持续优于强基线。

## 结语评价(编者)

**亮点**:

- 想法干净且对仗:OPD 的经典设定是「强模型教弱模型」,Seed 改成「看到事后信息的自己教瞎走的自己」——零外部依赖,老师永远跟学生同频;
- 「技能只存在于训练时」的设定非常工程友好:部署没有任何附加件(无分析器、无技能库、无检索),这是它与 Reflexion 类推理时反思方法的本质区别;
- 证据链完整:主结果、动力学、样本效率(60% 数据打全量)、泛化、消融、案例,且消融恰好证明「自进化」不是摆设(静态技能库掉最多);
- 全部用 3B/7B 小模型 + 单档资源完成,方法可复现性高。

**局限与疑点**:

- 技能质量依赖基座自身的分析能力——SFT 阶段还需要 GLM-5.2 这样的外部强模型做标注冷启动,更弱的基座能否自己转起来未验证;
- 基准集中在 ALFWorld/WebShop/搜索 QA 这类中等复杂度任务,对更长时程、工具链更复杂的 coding agent 场景(如 SWE-Bench)效果如何未知;
- 置信门基于「技能增强 vs 普通」的概率差,技能本身若含错误归因,门会放大它——论文未讨论技能错误率的影响。
