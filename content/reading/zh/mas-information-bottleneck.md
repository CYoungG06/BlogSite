## 摘要

LLM 驱动的多智能体系统(MAS)已成为复杂任务的一个有前景的范式。然而,它相对单智能体系统(SAS)的优势仍不明确,不同设定下表现忽好忽坏。本文提供一个信息瓶颈(information bottleneck,IB)视角来阐明 MAS 与 SAS 的差异。我们的关键观察是:SAS 把完整的推理轨迹积累在同一个共享上下文中,而 MAS 使用由有界中继消息连接的隔离局部上下文。我们证明:在中继带宽无限时,任何 SAS 都可以被一个「传输完整上游上下文」的 MAS 精确模拟。因此,MAS 的非平凡优势出现在**有界中继**下:压缩引入了根本性的权衡——削减冗余上下文可以提升效率,但也可能丢失任务相关信息。我们把这一权衡形式化为一个由有效参数 $\beta$ 控制的信息瓶颈:$\beta$ 刻画平衡如何随模型能力变化,并表明当上下文削减大于中继信息损失时,MAS 才有收益。我们在五个基准、三个模型规模上做了 18 组对照实验验证理论。结果表明:中继接近充分时 MAS 稳定占优(对弱模型尤其明显);而当中继造成信息丢失时,MAS 收益缩水甚至反转——尤其对已经能从冗余上下文提取有用信息的强模型。多智能体设计本质上是一个信息瓶颈优化问题,这一视角解释了有界 agent 间通信何时有益、何时有害。

## 1 引言

基于 LLM 的多智能体系统(MAS)已成为复杂任务的一个有前景范式:把任务分解为子任务、分派给专门的 agent,MAS 可以支持角色专门化、模块化推理和结构化协作,并在软件工程、数学推理、科学发现、规划等领域展现了实证收益。然而,MAS 相对 SAS 的优势仍不明确,不同设定下表现不一。近期研究显示:随着基座模型能力增强,MAS 的收益会减弱;或者收益其实来自更多算力而非架构优势。这些结果表明 MAS 并非天然更优,只在特定条件下有用。现有的比较研究多为实证,没有解释底层原因。这引出一个根本问题:**是什么样的结构性机制,决定了 MAS 何时优于 SAS?**

![图 1:按中继状态预测的 MAS 收益。当上下文削减大于能力加权的中继损失时 MAS 有益;更强的模型从压缩中获益更少、受信息丢失之害更多。](/images/reading/mas-information-bottleneck/relay_regime_3.png)

本文提出用**信息流视角**研究 SAS 与 MAS 的本质差别:差别可能不在 agent 的数量,而在信息如何组织与传输。SAS 维护一个共享上下文,所有子任务与中间推理轨迹都在其中累积;MAS 则使用由中继消息连接的隔离局部上下文。每个中继都是一个压缩接口:它能通过抑制噪声上下文减少下游干扰(好处),但也可能造成下游执行者无法恢复的信息丢失(代价)。这对立统一恰好被信息瓶颈原理刻画。从这个视角看,MAS 设计可以视为关于中继压缩的优化问题:如何在去掉上游无关上下文的同时,保留下游任务相关的信息?

我们证明,**在中继通信不受限时,MAS 与 SAS 等价**:MAS 只要传递完整上游上下文即可模拟 SAS。因此,非平凡的 MAS 效应来自有界中继压缩,而不仅仅来自任务分解。图 1 概括了我们框架的高层预测:当上下文削减大于中继信息丢失时 MAS 有益;当中继丢弃了下游相关信息时,收益缩水甚至反转。为解释这一权衡,我们把每个中继建模为一个信息瓶颈(图 2(b)),其有效参数 $\beta$ 刻画下游 LLM 的能力。由此得到分阶段的 MAS 收益:当上下文削减的好处超过能力加权的中继损失成本时,MAS 有益。随着基座 LLM 能力增强($\beta$ 增大),强模型从压缩中获益更少、对信息丢失更敏感。

我们用 18 组实证研究验证理论,覆盖五个 agentic 基准——ALFWorld、WebShop、WorkBench、WideSearch、TravelPlanner——和三个模型规模:Qwen2.5-7B、GPT-4o-mini、Qwen3.5-27B。如图 2(a)所示,我们精心设计了三个设定,把「分解」的作用与「基于中继的上下文隔离」分离开来。在全部基准和模型规模上:中继接近充分时 MAS 相对 SAS-contextflow 稳定提升;而当中继丢失下游相关信息时,收益减弱甚至反转,对强模型尤其如此。

**贡献总结**:

1. **MAS 的信息瓶颈框架**:把 MAS 设计形式化为隔离 worker 上下文之间的中继压缩;证明中继通信无界时任何 SAS 都可被 MAS 精确模拟——非平凡的 MAS 损益来自有界中继压缩。
2. **分阶段的 MAS 收益**:从 IB 目标推导逐阶段 MAS 收益——当上游噪声上下文削减的好处超过中继信息丢失的成本时 MAS 有益;有效参数 $\beta$ 刻画下游模型能力,解释了为什么同一个中继对弱模型有益、对强模型有害。
3. **受控实证验证**:五个基准、三个模型规模、18 组对比。通过比较 SAS、SAS-contextflow 与 MAS,我们把中继压缩与子任务分解分离开,证明观测到的 MAS 收益遵循预测的中继损失模式。我们的框架指出了如何设计更好的 MAS:优化中继,去掉上游无关上下文,保留下游相关信息。

## 2 相关工作

**多智能体系统。** 基于 LLM 的 MAS 把复杂任务分解给以自然语言通信的专门 agent。早期框架引入角色扮演、工作流执行与会话协议(MetaGPT、CAMEL、AutoGen),后续工作在软件开发、推理、科学发现与模拟上展示了收益。近期研究则考察 agent 数量增长时性能如何变化,既有提升也有非单调趋势。

**MAS 与 SAS 的实证研究。** 近期工作一致表明:MAS 的优势取决于任务与模型,且随基座 LLM 变强而减弱。与我们最接近的工作[han2025single]表明:在匹配思考 token 预算时,SAS 在多跳推理上可以追平甚至超过 MAS,作者把差距归因于通信瓶颈。这一观察与我们的中继信息丢失项 $\Delta_{i}(m_{i})$ 一致,但它只描述了 MAS 权衡的一面;我们的框架还刻画了上游上下文削减的好处 $H(M_{i}\mid m_{i})$,以及决定哪一项获胜的能力相关权重 $\beta$——从而把实证描述推进到「MAS 何时有益」的机理解释。

**信息瓶颈。** 信息瓶颈(IB)的思想把「压缩」与「保留相关性」的权衡形式化,已被用于研究深度表征学习与通信理论中的中继信道。本文是把它用于 LLM agent 系统的一次尝试:把每个 worker 间接口建模为中继瓶颈,并推导 MAS 收益的分阶段分解,以解释上述不一致的实证发现。

## 3 问题设定与 MAS–SAS 等价性

本节引入任务结构、agent 系统与 agent 间通信的形式化设定。我们把 agentic 任务建模为部分可观测的序贯决策过程,并用共同的基础 agent 抽象定义 SAS 与 MAS;然后给出无限带宽下的 MAS–SAS 等价,说明为什么压缩中继通信才是 MAS 损益的关键来源。

**Agentic 任务。** 形式化为部分可观测序贯决策过程:任务类 $\Gamma\triangleq\langle\mathcal{S},\mathcal{A},\mathcal{O},P,\Omega,\mathcal{X}\rangle$,其中 $\mathcal{S}$、$\mathcal{A}$、$\mathcal{O}$ 分别为状态、动作、观测空间,$P$ 为转移核,$\Omega$ 为观测核,$\mathcal{X}$ 为任务实例空间。

**基础 Agent。** 定义为 $\mathsf{Agent}\triangleq\langle\mathcal{T},\mathcal{M},\pi_{T},\pi_{A},\mathrm{Agg}\rangle$:$\mathcal{T}$ 为思考空间,$\mathcal{M}$ 为上下文空间,$\pi_{T}$、$\pi_{A}$ 为思考与动作策略,$\mathrm{Agg}$ 为输出函数。这里「上下文」指 agent 可见的累积执行历史,由之前各步的思考、动作与观测组成。给定任务实例 $X$ 与初始观测 $o_{0}$,agent 初始化 $M_{0}=(X,o_{0})$,并按 $M_{t+1}=M_{t}\oplus(\tau_{t},a_{t},o_{t})$ 更新上下文,其中 $\tau_{t}\sim\pi_{T}(\cdot\mid M_{t})$ 为思考,$a_{t}\sim\pi_{A}(\cdot\mid M_{t},\tau_{t})$ 为动作,$o_{t}$ 为结果观测。当 $\pi_{A}$ 发出终止动作时,agent 产出上下文 $M_{T}$ 与输出 $\mathrm{Agg}(M_{T})$。

**单智能体系统(SAS)。** 在任务实例 $X$ 上实例化一个基础 agent,初始上下文 $M_{0}=(X,o_{0})$,沿单条上下文轨迹产出 $M_{\mathrm{SAS}}$ 与最终输出 $y_{\mathrm{SAS}}=\mathrm{Agg}(M_{T})$。

**多智能体系统(MAS)。** 由一个规划器、$n$ 个 worker 组成的序列和一个系统级聚合器构成。规划器把实例 $X$ 分解为有序子实例序列 $X_{1},\ldots,X_{n}$;每个子实例 $X_{i}$ 诱导一个 worker 专属的目标变量 $Y_{i}$(worker $i$ 需要推断或产出的任务相关结果)。我们关注实例级分解:任务核 $(P,\Omega)$ 不变,仅把 $X$ 划分给各 worker。每个 worker 是运行在 $X_{i}$ 上的基础 agent;worker 内部按基础 agent 方式推进上下文,worker 之间则通过中继构成链:worker $i$ 产出中继 $m_{i}=\mathrm{Agg}_{i}(M_{i})$ 传给 worker $i+1$($m_{0}=\varnothing$),worker $i$ 以 $M_{0}^{(i)}=(X_{i},o_{0}^{(i)})\oplus m_{i-1}$ 初始化上下文。系统级聚合器再把所有终止记忆合成为最终输出 $y_{\mathrm{MAS}}=\mathrm{Agg}_{\mathrm{MAS}}(M_{1},\dots,M_{n})$。

**无限带宽 MAS–SAS 等价。** MAS 与 SAS 在两个结构维度上不同:规划器把任务 $X$ 分解为子任务 $X_{1},\dots,X_{n}$;worker 通过中继 $m_{1},\dots,m_{n-1}$ 通信。记 $B_{\mathrm{sys}}$ 为 worker 间带宽——单个中继 $m_{i}$ 能承载的最大熵,即 $H(m_{i})\leq B_{\mathrm{sys}}$。可以证明:当带宽无限增长时,任何 SAS 都能被 MAS 模拟(命题 3,证明见附录 A.1)。

但实践中 $B_{\mathrm{sys}}$ 是有界的:LLM agent 的通信通道受 token 限制,远小于 $H(M_{i})$,每个中继 $m_{i}$ 被迫成为上游上下文的压缩摘要。命题 3 因此说明:**中继压缩才是 MAS 相对 SAS 表现时好时坏的来源**——压缩有益还是有害,取决于中继相对下游子任务保留了什么、丢弃了什么。

## 4 作为信息瓶颈的 MAS 中继压缩

上文已证明:全上下文中继通信让 MAS 精确恢复 SAS。一个自然的问题是:中继有界时,什么决定了 MAS 的表现?本节表明,**中继设计是一个信息瓶颈优化问题**:中继通过移除上游噪声上下文带来好处,但丢失下游 worker 所需信息时就会有害。这一权衡导出了刻画 MAS 何时优于 SAS 的 MAS 收益。

### 4.1 作为信息瓶颈的中继压缩

考虑规划器分解下相邻的 worker $i$ 与 $i+1$。worker $i$ 持有上下文 $M_{i}$,产出中继消息 $m_{i}=\mathrm{Agg}_{i}(M_{i})$;worker $i+1$ 收到 $m_{i}$ 连同它的本地子任务 $X_{i+1}$(子指令与初始观测)。记 $Y_{i+1}$ 为该子任务的下游目标。由于 $m_{i}$ 压缩自 $M_{i}$,它携带的关于 $Y_{i+1}$ 的信息不可能超过 $M_{i}$:给定 $M_{i}$ 与 $X_{i+1}$ 后,$m_{i}$ 不提供关于 $Y_{i+1}$ 的额外信息(形式化论证见附录 A.2)。于是有马尔可夫关系:

$$ m_{i}\leftrightarrow M_{i}\leftrightarrow Y_{i+1}\mid X_{i+1}.$$

**MAS 设定下的 IB 目标。** 中继应当压缩上游上下文,同时保留下游子任务所需的信息。这一权衡可写成信息瓶颈目标:

$$ \min_{p(m_{i}\mid M_{i})}\;I(M_{i};m_{i})-\beta I(m_{i};Y_{i+1}\mid X_{i+1})$$

其中 $I(\cdot;\cdot)$ 为互信息。第一项衡量中继 $m_{i}$ 保留了多少上游信息,第二项衡量在给定本地子任务 $X_{i+1}$ 下保留了多少关于 $Y_{i+1}$ 的下游相关信息。该目标虽不被显式优化,但被 MAS 的设计(prompt、通信方式与模型能力)隐式诱导,刻画了最优中继。

**有效 $\beta$ 与 LLM 能力。** 在 IB 目标中,$\beta>0$ 控制压缩与预测信息保留之间的偏好。在 MAS 里,我们把 $\beta$ 解释为由下游 LLM 能力诱导的**有效参数**:弱模型对噪声上下文更敏感,偏好更强的压缩(有效 $\beta$ 较小);强模型能更好利用丰富上下文,更看重保留预测信息(有效 $\beta$ 较大)。这解释了为什么同一个中继可能对弱模型有益、却损害强模型。

**作为「不压缩中继」的 SAS。** 命题 3 表明,中继通信无界时 MAS 恢复 SAS。在 IB 形式下,这对应不压缩中继 $m_{i}=M_{i}$。代入式 (2):

$$ \mathcal{L}_{i}^{\mathrm{SAS}}(\beta)\triangleq\mathcal{L}_{i}^{\mathrm{MAS}}(m_{i}=M_{i};\beta)=H(M_{i})-\beta I(M_{i};Y_{i+1}\mid X_{i+1})$$

因此,在同一规划分解下,SAS 对应中继设计空间中的**不压缩点**,MAS 对应各种压缩中继设计。

### 4.2 MAS 收益:中继压缩中的权衡

IB 目标表明,MAS 中继必须平衡两种效应:压缩上游信息与保留下游相关信息。为分离第二种效应,定义**中继信息损失** $\Delta_{i}(m_{i})$:完整上游上下文 $M_{i}$ 被压缩为中继 $m_{i}$ 时,丢失了多少目标相关信息。由马尔可夫关系 (1) 有 $\Delta_{i}(m_{i})\geq 0$。其含义是:$\Delta_{i}(m_{i})=0$ 时,中继对下一个 worker 是**充分的**——它保留了 $M_{i}$ 中给定 $X_{i+1}$ 预测 $Y_{i+1}$ 所需的信息;$\Delta_{i}(m_{i})>0$ 时,中继不充分,值越大信息丢失越严重。

用中继信息损失重写 MAS 的 IB 目标:

$$ \mathcal{L}_{i}^{\mathrm{MAS}}(m_{i};\beta)=I(M_{i};m_{i})+\beta\Delta_{i}(m_{i})-\beta I(M_{i};Y_{i+1}\mid X_{i+1}).$$

最后一项与中继设计无关。

**MAS 收益。** 定义局部 MAS 收益为相对不压缩 SAS 基线的目标下降量:

$$ G_{i}^{\mathrm{MAS}}\triangleq\mathcal{L}_{i}^{\mathrm{SAS}}-\mathcal{L}_{i}^{\mathrm{MAS}}.$$

定理 4.2 表明,同一规划分解下的 MAS 收益由「上游上下文削减」与「中继信息丢失」的权衡决定:

$$ G_{i}^{\mathrm{MAS}}>0\quad\Longleftrightarrow\quad H(M_{i}\mid m_{i})>\beta\Delta_{i}(m_{i}).$$

其中 $H(M_{i}\mid m_{i})$ 衡量中继压缩移除的上游信息量(收益),$\beta\Delta_{i}(m_{i})$ 衡量下游相关信息的丢失(成本)。这一权衡是**能力依赖**的:弱模型从削减冗余上下文中获益更多;强模型更能利用冗余的上游信息,压缩的相对收益更小。综上,这一分解解释了 MAS 何时以及为何优于 SAS 的结构性机制。

**局部充分 ≠ 下游充分。** 上述收益分解是局部的,只针对单个中继接口。在串行 MAS 中,对 $Y_{i+1}$ 的局部充分并不自动意味着对更靠后目标的充分:只有当 $M_{i}$ 中 $Y_{j}$ 所需的全部信息都已被下一目标 $Y_{i+1}$ 捕获时,局部充分的中继才对 $Y_{j}$($j>i+1$)也充分;否则,若 $Y_{j}$ 仍依赖 $M_{i}$ 中未反映在 $Y_{i+1}$ 里的细节,隐藏的长程依赖会让局部充分的中继对后面的 worker 不充分(详见附录 A.3)。

> **译注**:这套形式化最有解释力的一点是 $\beta$ 的解读——它不是调出来的超参,而是「下游模型能力」的影子变量。业界今年的经验现象「模型越强,MAS 相对 SAS 的优势越小」在这里变成一个推论:强模型 β 大,信息丢失被放大、压缩收益被缩小。把「该堆几个 agent」从玄学变成了可以推理的问题。

## 5 实验

### 5.1 实验设置

**基准。** 在五个覆盖不同任务结构与中继复杂度状态的 agentic 基准上验证:ALFWorld、WebShop、WorkBench、WideSearch、TravelPlanner。我们用 $\delta$ 表示每个基准的估计**中继复杂度**——中继做到充分有多难的代理量。对 worker $i$ 到 $i+1$ 的中继,记 $m_{i}^{\star}$ 为局部充分中继,定义其分阶段复杂度:

$$\delta_{i}\triangleq\min_{m_{i}}L(m_{i})\quad\text{s.t.}\quad\Delta_{i}(m_{i})=0,$$

其中 $L(\cdot)$ 为中继描述长度,$\delta$ 即「下游充分中继的最短描述长度」,基准级 $\delta$ 取各子任务的平均。按任务信息结构给每个基准划定预测 $\delta$ 状态:$\delta\approx 0$ 表示下游相关信息可以被紧凑描述;$\delta>0$ 表示需要详细的上游证据;$\delta\gg 0$ 表示充分性需要大量精确的任务特定细节(如先前状态、命令输出、执行历史)。TravelPlanner 在同一任务里同时包含两种状态:常识检查(评估局部阶段合理性,低复杂度)与硬约束(要求全局计划一致性,高复杂度),因此分别报告 TravelPlanner-CS($\delta\approx 0$)与 TravelPlanner-HC($\delta>0$)。指标:ALFWorld 与 WorkBench 报成功率(SR),WebShop 报 SR 与平均奖励,WideSearch 报 Item F1 与 Row F1,TravelPlanner 报 SR、常识与硬约束宏通过率。

**原型。** 为把规划器引入的子任务结构与中继压缩分离开,比较三个受控原型(图 2(a)):**SAS** 标准单智能体,一个 agent 接收原始指令,把全部中间推理、动作与观测累积在同一上下文中直到产出输出;**SAS-contextflow** 单智能体但遵循与 MAS 相同的规划器子任务序列与角色分配,同时保持累积上下文全程可见——每一步都以当前子指令加完整上游上下文为条件;**MAS** 使用与 SAS-contextflow 相同的子任务序列,但每个子任务交给一个拥有隔离局部上下文的 worker,worker $i$ 只收到子指令与从上游上下文压缩来的中继 $m_{i-1}$。因此 MAS 与 SAS-contextflow 的唯一差别,就是把「完整累积上下文」换成「压缩中继通信」——恰好对应上下文削减与中继信息丢失的权衡。三个原型在相同的最大总步数预算下评估,保证公平。

**模型。** 三个能力档:Qwen2.5-7B-Instruct(弱)、GPT-4o-mini(中)、Qwen3.5-27B(强,AWQ 4bit 量化)。这条能力轴用来检验同一个中继复杂度状态如何与模型能力相互作用。

实验围绕两个研究问题验证定理 4.2:**(RQ1)** 低复杂度中继($\delta\approx 0$)是否如预测般,相对 SAS-contextflow 产生稳定的 MAS 收益?**(RQ2)** 高复杂度中继($\delta>0$)是否在有界通信无法保留下游相关信息时,导致 MAS 收益缩水或反转?

### 5.2 主要结果

![图 3:三个模型规模下的逐任务表现(能力从左到右递增)。绿色/红色阴影分别表示 MAS 优于/劣于 SAS-contextflow 的区域。](/images/reading/mas-information-bottleneck/x2.png)

图 3 画出 MAS、SAS-contextflow 与 SAS 随模型能力变化的轨迹,绿色为 MAS 领先区、红色为落后区。表 2(原文)按估计中继复杂度 $\delta$ 归类并报告实证趋势,详细数值见原文附录 C.1。以下按中继复杂度状态组织分析。

**RQ1:低复杂度中继基准($\delta\approx 0$):ALFWorld、WideSearch、TravelPlanner-CS。** 这三个基准的自然分解允许紧凑的充分中继:WideSearch 的子查询并行且基本独立,几乎不需要跨 worker 中继;ALFWorld 的 MAS 设计由探索者定位目标物体、执行者完成动作序列,中继只需提供物体位置;TravelPlanner-CS 评估局部常识可行性(交通、住宿、餐饮、景点合理性)。结果显示,MAS 在这些基准上与 SAS 相当或更强;更重要的是,在与 SAS-contextflow 的受控对比中,MAS 取得**稳定的正收益**——ALFWorld:$+0.194$、$+0.157$、$+0.023$;WideSearch(Item F1):$+0.079$、$+0.063$、$+0.028$;TravelPlanner-CS:$+0.011$、$+0.183$、$+0.028$(依次为 Qwen2.5-7B、GPT-4o-mini、Qwen3.5-27B)。这支持低复杂度中继的预测:当下游相关信息能被紧凑传输时,MAS 通过减少上下文干扰获益;同时,更强的模型从这种隔离中获得的边际收益更小(收益随能力递减)。

**RQ2:高复杂度中继基准($\delta>0$):WorkBench、WebShop、TravelPlanner-HC。** 这些基准的下游决策需要详细的上游证据,有界中继无法完整保留:WorkBench 要求先跨域检索证据再执行写操作,后续决策依赖精确的中间读取结果(对象 ID、时间戳、姓名-邮箱解析),中继充分性非常复杂($\delta\gg 0$);WebShop 的「搜索后选择」分解允许中继摘要候选商品,但最终选择仍依赖细粒度属性、价格、过滤条件与导航历史($\delta>0$);TravelPlanner-HC 的硬约束要求跨子任务的全局行程一致性,同属高复杂度。结果显示:MAS 相对 SAS 不再一致更强,且最强的模型往往更偏好完整共享上下文(尤其 TravelPlanner-HC)。受控对比下:**WorkBench 在三个模型上均为负收益**($-0.005$、$-0.086$、$-0.014$),严重的中继信息丢失主导了权衡;**WebShop 与 TravelPlanner-HC 出现能力依赖的反转**——WebShop 从弱、中模型的 $+0.080$、$+0.086$ 变为强模型的 $-0.003$;TravelPlanner-HC 从 $+0.017$、$+0.161$ 骤变为 $-0.233$。这些「由正转负」支持定理 4.2:弱模型受益于噪声上下文削减,而强模型能更好利用完整上下文,高复杂度有界中继造成的信息丢失最终占了上风。

总体而言,结果支持估计的中继状态模式:充分中继下 MAS 收益普遍为正、但随模型能力递减;不充分中继要么持续为负,要么随模型变强由正转负。**MAS 在上下文削减占主导时有益,在中继信息丢失成为限制因素时失去优势——尤其在全局耦合的任务上。**

## 6 消融研究

两组消融检验支配 MAS 收益的核心机制:中继带来的上下文隔离(区别于分解结构),以及中继充分性(区别于中继长度)。

**仅规划结构能否解释 MAS 收益?** 引入 SAS-Plan 原型:单智能体接收与 MAS worker 相同的分解子指令,但在单一共享上下文上执行。按第 4 节的形式化,SAS、SAS-Plan、SAS-contextflow 都位于中继设计空间的不压缩点($H(M_{i}\mid m_{i}){=}0$,$\Delta_{i}{=}0$)。结果显示,SAS-Plan 在 ALFWorld 上相对 SAS 一致更差(三个模型分别 $-0.068$、$-0.059$、$-0.194$);在 WebShop 与 TravelPlanner 上也普遍低于标准 SAS,且边际提升远小于 MAS。这确认了:**上下文隔离(而非分解结构本身)才是 MAS 收益的起效机制**。

**中继信息丢失是否因果性地支配表现?** 定理 4.2 预测 $\Delta_{i}(m_{i})$ 是直接的表现成本。构造更高损失的中继变体:故意移除下游所需信息而保持 MAS 分解不变——ALFWorld 中移除位置与物体状态信息;WebShop 中只保留商品名,丢弃价格、属性与导航路径。结果:加大中继信息丢失在所有模型上都一致降低表现。这为「$\Delta_{i}(m_{i})$ 是真实成本而非描述性量」提供了因果证据:当中继被故意省略下游必需信息时,表现随之下降,与信息瓶颈「压缩只在保留任务相关信息时才有益」的预测一致。

## 7 总结与局限

**总结。** 本文提出一个信息瓶颈框架,解释基于 LLM 的多智能体系统何时优于单智能体系统。核心见解:MAS 的损益来自有界的 worker 间中继——它在上下文削减与下游相关信息丢失之间权衡;有效参数 $\beta$ 刻画该权衡如何随模型能力变化,解释了为什么同一个中继能助弱模型、却伤强模型。五个基准、三个模型规模的实验支持了这一预测:中继接近充分时 MAS 有益,中继高度不充分时优势消失。

**启示。** 框架把 MAS 设计变成两个实际问题。其一,**什么时候该用 MAS?** 当任务允许低中继信息丢失的分解时,MAS 收益直接来自上游上下文削减。其二,更常见的情形下,**中继损失不可避免时该如何设计 MAS?** IB 目标指出了自然的下一步:把中继编码器与下游任务联合优化。由能力相关的 $\beta$ 引申出的更长远问题是:自适应中继能否在由不同基座模型组成的异构 agent 团队中维持 MAS 收益。

**局限。** $\beta$ 目前只是定性的能力参数,而非可直接估计的量;$\delta$ 也是靠任务结构分析划定,而非定量估计。未来工作需要为两者开发定量估计器,并用它们训练自适应中继。

## 结语评价(编者)

**亮点**:

- 把「MAS 有没有用」从现象升级为机理:无限带宽等价(命题 3)干净利落,直接锁定了非平凡性的来源就是有界中继;
- $\beta$ 的能力解读解释了一整年的业界实证困惑(为什么强模型上 MAS 优势消失),这是论文最值钱的一个洞见;
- 实验设计讲究:SAS-contextflow 与 SAS-Plan 两个对照把「分解」与「上下文隔离」拆干净,消融里还有少见的因果性验证(故意污染中继);
- 工程启示直接:先估任务的 $\delta$(中继做到充分有多难),再决定用不用 MAS——全局耦合强的任务(行程规划、跨域检索)天生不适合。

**局限与疑点**:

- $\beta$ 与 $\delta$ 都还是定性概念,没有可计算的估计器——框架能解释,但还不能直接指导调参;离「给定任务自动算一下该不该用 MAS」还有距离;
- 实验只覆盖串行链式分解,对更复杂的拓扑(并行扇出、层级、辩论式)是否仍成立未验证;
- 三个模型里「强模型」也只到 27B,前沿旗舰上的 $\beta$ 曲线形态未可知。
